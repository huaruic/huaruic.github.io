# 博客文章创建和发布指南

## 方式一：使用 Hugo 命令创建（推荐）

### 1. 创建中文文章

```bash
# 创建一篇新的中文博客文章
hugo new blog/zh/文章名称.md

# 例如：
hugo new blog/zh/my-new-post.md
```

### 2. 创建英文文章

```bash
# 创建一篇新的英文博客文章
hugo new blog/en/文章名称.md

# 例如：
hugo new blog/en/my-new-post.md
```

## 方式二：手动创建文件

直接在 `content/blog/zh/` 目录下创建 `.md` 文件，使用以下模板：

```markdown
---
title: 文章标题
date: 2025-01-01T12:00:00+08:00
tags: ["标签1", "标签2"]
featured: true
---

文章摘要部分（会显示在列表中）

<!--more-->

这里是正文内容...
```

### Front Matter 字段说明：
- `title`: 文章标题
- `date`: 发布日期（格式：YYYY-MM-DDTHH:mm:ss+08:00）
- `tags`: 标签列表
- `featured`: 是否精选（true/false）
- `series`: 系列文章（可选）

## 本地预览

```bash
# 启动本地服务器预览
hugo server -D

# 然后在浏览器打开 http://localhost:1313/
```

## 发布到 GitHub

### 1. 检查状态

```bash
git status
```

### 2. 添加文件

```bash
# 添加单个文件
git add content/blog/zh/你的文章.md

# 或者添加所有更改
git add .
```

### 3. 提交更改

```bash
git commit -m "新增文章：文章标题"
```

### 4. 推送到远程仓库

```bash
git push origin master
```

## 完整示例流程

```bash
# 1. 创建新文章
hugo new blog/zh/my-awesome-post.md

# 2. 编辑文章内容（使用你喜欢的编辑器）

# 3. 本地预览
hugo server -D

# 4. 确认无误后，添加并提交
git add content/blog/zh/my-awesome-post.md
git commit -m "新增文章：我的优秀文章"

# 5. 推送到 GitHub
git push origin master
```

## Hugo 生成 HTML 和自动部署

### ⚠️ 重要：你只需要推送 Markdown 文件！

**你不需要手动生成 HTML，也不需要推送 `public` 目录。**

项目已经配置了 **GitHub Actions** 自动部署：

1. **自动构建**：当你 push markdown 文件到 `master` 分支后，GitHub Actions 会自动：
   - 安装 Hugo
   - 运行 `hugo --minify` 命令生成 HTML 文件到 `public` 目录
   - 将生成的 HTML 部署到 `gh-pages` 分支

2. **`.gitignore` 配置**：`public/` 目录已经在 `.gitignore` 中被忽略，所以你不需要（也不应该）推送它

3. **工作流程**：
   ```
   编辑 markdown → git push → GitHub Actions 自动构建 HTML → 自动部署到网站
   ```

### 查看部署状态

推送后，你可以：
- 访问 GitHub 仓库的 **Actions** 标签页查看构建状态
- 构建成功后，网站会在几分钟内更新

### 本地生成 HTML（仅用于测试）

如果你想在本地测试构建：

```bash
# 生成静态网站到 public 目录
hugo --minify

# public 目录会被创建，包含所有 HTML 文件
# 注意：不要提交 public 目录，它会被自动生成
```

## 注意事项

1. **文件名建议**：使用英文和连字符，例如 `my-new-post.md`
2. **日期格式**：确保日期格式正确，时区使用 `+08:00`（中国时区）
3. **标签**：使用数组格式，例如 `["工作", "生活"]`
4. **文章摘要**：`<!--more-->` 之前的内容会作为摘要显示在列表中
5. **不要推送 public 目录**：`public/` 已在 `.gitignore` 中，会自动生成
